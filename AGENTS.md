# GotheWord 开发准则

本文件适用于整个仓库。所有开发者和编码代理在新增、修改或审查代码时都必须遵守以下规则。

## UI 组件唯一来源

- 所有界面组件必须使用 [`animal-island-ui`](https://github.com/guokaigdg/animal-island-ui) 提供的组件。
- 禁止自行创建、复制或仿制按钮、输入框、卡片、弹窗、菜单、提示、标签、进度条等通用 UI 组件。
- 禁止引入其他 UI 组件库，也禁止使用原生 HTML 元素配合样式重新实现 `animal-island-ui` 已有的组件。
- 页面和业务模块只能组合 `animal-island-ui` 的现有组件；不得为了统一参数或样式而额外封装一层本地通用组件。
- 原生元素仅可用于库中没有对应职责的语义结构或纯布局容器，且不得借此复刻通用 UI 组件。

## 使用要求

- 安装依赖时使用包名 `animal-island-ui`。
- 从包入口直接导入组件，并在应用入口全局引入样式：

  ```tsx
  import { Button, Card } from 'animal-island-ui';
  import 'animal-island-ui/style';
  ```

- 实现界面前，先检查该库的 README、`AI_USAGE.md`、在线文档和导出类型，确认是否已有满足需求的组件及正确 API。
- 优先使用组件公开的属性、变体和设计能力。除修复明确的兼容问题外，不覆盖组件内部样式，不复制组件源码，不依赖未公开的内部实现。
- 不得臆造组件名、属性或事件；实现和审查时必须以当前安装版本的类型定义或官方文档为准。

## 组件缺失时的处理

当 `animal-island-ui` 没有满足需求的组件时：

1. 停止该部分实现，不得以临时组件、自定义样式、第三方组件库或硬编码方案绕过。
2. 记录缺失的交互、视觉和无障碍要求，并向需求方说明库中没有可用组件。
3. 由需求方决定调整设计、等待上游组件，或先向 `animal-island-ui` 提交组件需求。未得到明确同意前，不在本仓库新增 UI 组件。

## 变更检查

每次涉及界面的变更在交付前必须确认：

- 新增界面元素均来自 `animal-island-ui`，且导入路径正确。
- 没有新增本地通用 UI 组件、其他 UI 组件库或对现有组件的重复实现。
- `animal-island-ui/style` 已在应用入口引入，页面未重复引入或复制其样式。
- 使用的组件 API 与当前安装版本一致，并完成相关类型检查、测试或构建验证。
